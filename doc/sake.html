3 Use

Usage: 

sake [-h] [-file <sake-file>] [initial-task]
  • -h: help page
  • -f <sake-file>: assign the sakefile to load (defaults to sakefile.scm)
  • <initial-task> the task that have to be run (defaults to build)

4 Sakefile language

4.1 Parameters

4.1.1 current-build-directory    parameter

(current-build-directory)           ; get parameter value
(current-build-directory <newval>)  ; set parameter

defaults to $PWD/build (i.e. if run in directory foo/bar it will be foo/bar/build defines the default directory where compiled files will be put. (can be used with parameterize, see gambit-c parameters

4.1.2 current-source-directory    parameter

(current-source-directory)           ; get parameter value
(current-source-directory <newval>)  ; set parameter

defaults to $PWD/sources (i.e. if run in directory foo/bar it will be foo/bar/source defines the default directory where source files exists

4.1.3 current-project-name    parameter

(current-project-name)           ; get parameter value
(current-project-name <newval>)  ; set parameter

defaults to the name of $PWD (i.e. if run in directory foo/bar it will be bar) defines project name. My rule of the thumb is to have a project composed by multple libraries loadable as (load "~~<project-name>/<module-name>")

4.1.4 current-module-name    parameter

(current-module-name)           ; get parameter value
(current-module-name <newval>)  ; set parameter

defaults to (current-project-name) can be used to build various modules that refers to the same project.

4.2 Tasks

4.2.1 task    macro

(task (<depends> ...) <body> ...)

creates a new task, that depends from (<depends …). The body of the task is what can be in a lambda form

4.2.2 define-task    macro

(define-task <name> (<depends> ...) <body> ...)

shorthand for (define <name> (task (<depends> …) <body> …))

4.2.3 task-run    macro

(task-run <task>)

run the task, and the dependant tasks before if needed

4.3 Files

4.3.1 fileset :function:    macro

(fileset dir: <dir>
         test: <test>
         recur: <bool>)

creates a list containing the names of the files present in directory dir. the optional test: keyword let's you express a test function to which the file name is passed, and must returns true to be part of the resulting list.

4.3.2 make-directory    function
